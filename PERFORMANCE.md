# Performance log

Measured performance changes, one row per PR, in master commit order, newest first. Append a row in
the same PR that changes performance, with the measured impact and an evidence link. Numbers in a
row come from that row's paired evidence, and paired values read before → after; rows are measured
under different conditions and are not comparable to each other.

## Verification (solve) performance

Changes to the time it takes to verify a sample: building the MIP model, tightening ReLU bounds
(interval arithmetic, LP, and MIP solves), and the main solve that proves robustness or finds an
adversarial example. This work scales with the number of samples verified. Columns, measured on the
paired 500-sample WK17a LP benchmark:

- **Total**: end-to-end wall clock summed over samples where both runs built a solver model.
  Semantic-outcome and solve-status changes stay included because they are part of the measured
  workload; row notes call them out explicitly.
- **Median ratio**: per-sample after ÷ before over the same paired samples; below 1 is faster.

rows overlap in their before/after runs, so attribution between them is approximate.

  different set of jobs, so values are not comparable between rows.
